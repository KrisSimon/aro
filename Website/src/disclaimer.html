Important information about ARO and its intended use.

⚠

Experimental Project

ARO is an experimental programming language created for research and learning purposes. It is not intended for production use.

🧪 Experimentation Only

ARO is designed purely for experimentation, learning, and exploring new ideas in programming language design. It serves as a proof-of-concept for AI-native language development and Feature-Driven Development (FDD) principles. Use it to learn, experiment, and have fun—but not for anything that matters.

🚫 Not for Production

This language is explicitly not intended for production use. It lacks the security hardening, thorough testing, performance optimization, and stability guarantees required for production systems. Do not use ARO to build applications that handle real user data, financial transactions, or any mission-critical operations.

🤖 100% AI-Generated Code

Every single line of code in this project—the compiler, runtime, documentation, and this website—has been generated entirely by AI. This includes the parser, semantic analyzer, execution engine, HTTP server, and all supporting infrastructure. The entire codebase is the product of AI-assisted development.

👀 No Human Review

The AI-generated code has not undergone formal human code review. While the code functions and demonstrates the intended concepts, it has not been audited for security vulnerabilities, edge cases, or production-readiness. There may be bugs, security issues, or unexpected behaviors that human review would typically catch.

✅

What ARO IS Good For

Learning about language design, exploring AI-native programming concepts, understanding FDD principles, experimenting with new ideas, and having fun with code.

❌

What ARO IS NOT Good For

Production applications, handling sensitive data, financial systems, healthcare applications, or any use case where reliability, security, or correctness is critical.
